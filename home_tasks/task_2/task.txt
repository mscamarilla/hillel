Домашнее задание #2

Переменные. Основные элементы управления. Функции.
1. Перечитать все по ссылкам которые в комментариях под занятиями в lms(Если вы еще этого не сделали) и повторить примеры кода которые там приводятся.
2. Создать массив с ключами: name, surname, age, gender, каждый из которых принимает значение null. Написать функцию, которая принимает входящий параметр массив и по ключам устанавливает значения. Функция результат не возвращает. После вызова функции вывести массив с готовыми значениями на экран.
ПС: вспомнить про области видимости и передачи параметров в функцию по значениям и указателям.
3. Функция. Принимает один входящий параметр - массив. Считает сумму всех элементов массива и возвращает результат. В функции должна присутствовать валидация для каждого элемента массива(если, например, один из элементов содержит строку или что-то, что не является числом, то принимать за ноль). Результат работы функции вывести на экран.
ПС: Как определить что переменная - число, спросите в интернетах.
4. Написать функцию которая принимает порядковый номер дня недели (1-7) и выводит на экран сообщение типа: “Hello. Today is Monday.” или “There are only 7 days in a week!“ - если в функцию передано что-то кроме 1-7.
Использовать конструкцию if-else.
5. Функцию из третьего задания реализовать еще двумя способами:
- с использованием оператора switch(найти документацию по использованию в интернетах)
- и с использованием в функции предопределенного массива содержащего 7 элементов в которых ключ - порядковый номер недели, значение - строка с именем дня недели. Логика функции должна определять существует ли в массиве ключ с полученным значением (1-7) и так же выводить сообщение.
ПС: Как определить есть ли ключ в массиве - спросите в интернетах.
6. Мы не успели обсудить на уроке понятие рекурсивного вызова функций, потому ознакомьтесь самостоятельно и примените при реализации функции вычисления факториала числа. Функция принимает один параметр, валидирует его, возвращает значение. Результат работы функции вывести на экран.
(Такое задание или подобное часто встречается на собеседованиях.)
